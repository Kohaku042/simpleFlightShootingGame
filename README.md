## 1. シンプルなフライトシューティングゲーム
本制作物は、TypeScriptとHTML、CSS、およびThree.jsを組み合わせて制作した3Dのフライトシューティングゲームである。

## 2. 実装における工夫と努力した点

### ジェットエンジンの噴射エフェクト
戦闘機のジェット噴射エフェクトは、当初パーティクルシステムでの実装を予定していた。しかし、処理負荷の増加と実装コストの観点から方針を変更し、透明度を持たせたメッシュ（ConeGeometry）のスケールをランダムかつ小刻みに変化させる手法で実装した。これにより、軽量でありながらも躍動感のある青い炎の揺らぎを表現している。

### 航空機らしい挙動の再現（失速）
単なる移動だけでなく、飛行機らしい物理的な挙動を出すために「失速」の仕組みを適用した。機体が設定された限界高度に達した際、強制的に機首が下がり、高度が落ちる処理を組み込んでいる。

### 天候エフェクト（雪）の最適化
冬の山岳地帯を表現するため、雪のエフェクトを追加した。当初は空間全体にランダムに雪のパーティクルを発生させていたが、処理負荷が大きくなる問題があった。これを解決するため、カメラの周囲（見えている範囲）にのみ雪が降るエフェクトへと変更し、カメラの後方にいったパーティクルを前方にワープさせることで、処理を少なくしつつ吹雪の中を飛んでいるような表現を実現した。

### 地形生成と描画の工夫
`simplex-noise.js` のPerlinノイズを利用し、無限に続く山岳地形をプロシージャルに生成した。
さらに、遠景の地形チャンクが生成される際の不自然なポップインをごまかすため、`THREE.Fog` を使用し、空間に霧をかけることで自然な奥行きとフェードインを演出した。

### 3Dモデルの読み込みと当たり判定
`public` フォルダに用意して読み込みを行った。本制作物ではより実践的でモダンな拡張子である `.glb` を使用して戦闘機モデルを読み込むことにした。
また、敵機や地形との衝突判定には、`THREE.Box3` を使用し、正確なバウンディングボックスによる当たり判定を実装した。

### 実装を断念した点
敵を倒した際や、地面に衝突した際の爆発エフェクト（パーティクル）を追加したかったが、時間的に間に合わないため実装を断念した。今後の課題としたい。

## 3. 生成AIの利用について

本制作物では、開発の補助として生成AIを利用した。ただし、AIが生成したコードはかなりの頻度で間違っていたため、最終的には大部分を手で修正・調整している。

**1. 戦闘機の3Dモデル生成**
- **使用モデル**: Hunyuan3D-2.1
- **対応・修正**: AI生成モデルにいくつか問題点があったため、Blenderを用いて手動で修正を行った。編集モードでの頂点の移動・削除・追加、前輪の追加、スカルプトモードでの凹凸修正を行い、テクスチャの調整も自ら行った。

**2. 機体の回転描画の処理**
- **使用モデル**: Gemini 3.1 Pro
- **対応・修正**: クォータニオンを用いた複雑な回転処理のベース作成に使用。自らコードを解読し、理解のためのコメントアウトを追加した。

**3. プロシージャル生成の処理、メモリリーク防止**
- **使用モデル**: Gemini 3.1 Pro
- **対応・修正**: 地形チャンクの動的生成および破棄の処理において利用。仕組みを理解し、自ら理解のためのコメントアウトを追加した。

**4. コードの最適化、冗長な処理の編集**
- **使用モデル**: Gemini 3.1 Pro
- **対応・修正**: 提案されたコードを目視による確認で取捨選択し適用した。わからなかった部分には、調べてコメントアウトを追加した。

**5. 命名の最適化**
- **使用モデル**: Gemini 3.1 Pro
- **対応・修正**: 変数名や関数名のリファクタリングに利用し、すべて目視による確認を行った。
